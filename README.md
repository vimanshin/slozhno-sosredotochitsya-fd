## Процесс работы

Сначала сверстал тёмную тему на mobile, затем добавил tablet и desktop
через media queries, после этого — светлую тему и финальную полировку.

**Подход:**
- **mobile-first** — базовые стили под 375px, брейкпоинты 768px и 1024px;
- сверка с макетом в Figma (Pixel Perfect) на всех разрешениях и в обеих темах;
- вёрстка в отдельной ветке `lightcss`, затем слияние в `main`;
- финальная проверка: валидатор HTML, Stylelint, Tab-навигация, отсутствие
  горизонтального скролла.
**Структура стилей:** переменные в `variables.css`, глобальные стили в `globals.css`,
основная вёрстка в `style.css`, темы — в `dark.css` и `light.css`.

### Стек технологий

- **HTML5** — семантическая разметка (`header`, `main`, `section`, `footer`, `nav`)
- **CSS3** — Flexbox, CSS Grid, media queries, псевдоклассы (`:hover`, `:focus-visible`)
- **CSS Custom Properties** — цвета и шрифты через переменные, переопределение в темах
- **БЭМ** — именование классов (`block__element`)
- **Адаптивность** — mobile-first, резиновая вёрстка между брейкпоинтами
- **Две цветовые темы** — классы `theme-light` / `theme-dark` + `prefers-color-scheme`
- **Инструменты:** Git, Prettier, Stylelint, W3C HTML Validator, Pixel Perfect

*JavaScript для вёрстки не писал — переключение тем уже было в стартовом `script.js`.*

### Чему я научился

- Строить адаптивный лейаут с **Grid** и **Flexbox** под три разрешения экрана
- Организовывать CSS по файлам и не дублировать стили (группировка общих селекторов)
- Делать **две темы** через CSS-переменные и отдельные файлы, а не копированием стилей
- Работать с **`:focus-visible`** для доступной клавиатурной навигации
- Сверять вёрстку с макетом по допускам (автотесты и ревью)
- Писать валидный и семантический HTML (один `h1`, правильная иерархия заголовков)
- Вести проект поэтапно: план → вёрстка → проверка → рефакторинг → сдача
